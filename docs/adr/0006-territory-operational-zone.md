# ADR-0006: Territory — операционная зона между Project и OperationalLocation

**Status:** accepted
**Date:** 2026-05-26

## Context

«Регион» жил в системе текстом: в адресе `operational_locations.name` (`143005, Московская обл, ...`) и продублированно в названиях проектов — «Магнит» разрезан на 6 проектов по регионам (плюс точные дубли вроде Сочи/Краснодар). «1 СВ = 1 регион» нигде не смоделировано.

Следствия: дубли проектов-клонов, отчёты «по региону» = парсинг строк, нет места для среднего звена (супервайзер / региональный руководитель). Project-scoped регион сам по себе бессмыслен, пока проекты нарезаны по регионам — выигрыш только если новый слой позволяет проекты схлопнуть.

## Decision

Вводим **Territory** — Core-сущность (canonical), project-scoped, слой между Project и OperationalLocation: `operational_locations.territory_id`. Проекты-клоны, нарезанные по регионам, консолидируются в один проект (6 «Магнит …» → 1 «Магнит» + 6 территорий под ним). Территория — плоская операционная зона (грань переменная: где область, где город), с nullable-тегом `federal_subject` для гео-роллапа. Контрагента отдельной сущностью **не** вводим.

## Alternatives

- **Глобальный гео-регион (субъект РФ)** — чистая география, но ломает «СВ ведёт свою группу точек» и «1 СВ = 1 зона» (Кемерово+Новокузнецк = один субъект, две зоны). Гео добираем тегом `federal_subject`.
- **Оставить регион в `project.name`** — дубли проектов, отчёты строками, дрейф name/адрес.
- **Контрагент как сущность** — излишество сейчас; `project = бренд` это и так покрывает.
- **Имя «регион» / «unit»** — «регион» врёт (зона ≠ субъект РФ); `unit` конфликтует со складским `sku.unit` (единица измерения). Взяли Territory — отраслевой термин field-sales.

## Consequences

**Получаем:** один проект вместо шести клонов (нет дублей-проектов); массовое управление группой точек в отчётах и операционных сценариях; дом для среднего звена (`territory_assignments`: supervisor / regional_director); гео-роллап через `federal_subject`; роли получают грань (проект / территория / точка).

**Платим:** data-миграция в окне — территория парсится из адреса `operational_locations.name` (НЕ из `rusaifin.city`, он загрязнён названиями проектов / реф-ссылками); слияние дубль-проектов + перепривязка точек / memberships / assignments + маппинг rusaifin. Новый слой читается складом и rusaifin после reader-switch.

## Revisit if

- Понадобится одна территория на несколько проектов — пересмотреть project-scoped в сторону m2m.
- Понадобится полноценный справочник субъектов РФ — сейчас это nullable-тег, не сущность.
