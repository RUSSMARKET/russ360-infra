## Симптом
При вводе списка точек строкой без пробела после запятой (`"10,20,30"`) оформляется только первая точка (10), остальные молча игнорируются; пользователь видит «Роль успешно установлена».

## Доказательства (file:line)
- `rusaifin/app/Http/Requests/Staff/SetRegistrationRoleRequest.php:28-38` — `prepareForValidation`: `explode(', ', $this->points_id)` (разделитель строго «запятая+пробел»). Для `"10,20,30"` вернёт `["10,20,30"]`, далее `(int)"10,20,30" === 10` → массив из одной точки.
- Тест `tests/Feature/AgentTestPeriodManagementTest.php:32` шлёт `points_id` массивом `[1]`, минуя эту ветку → баг не покрыт.

## Триггер / repro
Клиент отправляет `points_id` строкой с разделителем `,` без пробела.

## Корневая причина (гипотеза)
Хрупкий парсинг по литералу `', '` вместо нормализованного разделителя.

## Радиус поражения
Только строковый ввод `points_id`. Если фронт всегда шлёт массив или строго `', '` — не проявляется (отсюда needs-verification: требуется проверить фактическую сериализацию фронта).

## Направление фикса (1-2 строки, НЕ реализовано)
`preg_split('/\s*,\s*/', trim($this->points_id))` с фильтром пустых, либо принимать только массив.
