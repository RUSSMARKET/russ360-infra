# Ревью модуля учёта карт ОТП — 2026-06-23

Источник: `/product-analysis` (4 Explore-агента: backend / frontend-UX / архитектура / бизнес-gaps),
сведено и перепроверено по коду. Рабочий план закрытия — по этому файлу, блок за блоком.

## Резюме
Коробочная модель (ADR-0007 add.#2) реализована корректно и целостно: цикл
приёмка→отправка→приём→инвентаризация→возврат→отчёт работает, инварианты держатся,
18 тестов зелёные. Под главную бизнес-цель (−5 М₽/мес потерь) не хватает трёх вещей:
трассировки потери по агенту/точке выдачи, сверки с реестром банка, контроля
регулярности инвентаризации.

## Кросс-валидация (отброшенные ложные находки агентов)
- backend-агент: `isBoundToLocation` зовёт `isEmployeeAssignedToLocation(core_location_external_id)` —
  **неверно**, в коде `core_employee_external_id`. Не баг.
- проверить наличие enum `CardState` перед действиями (в коде `CardBatch::STATE_*` строковые
  константы, не enum) — находка агента C про enum может быть неточной.

## ✅ Сделано
- Коробочная модель целиком: `card_batches`(коробка), `card_inventories`(+коды открытой), `card_dossier_returns`. FSM `in_central→in_transit→at_point→depleted` + `cancel`.
- Операции: приёмка, отправка чек-листом, приём на точке, отмена партии, еженедельная инвентаризация, возврат досье (идемпотентно `partner+code`).
- Отчёт «Отчёт» (штуки, без денег): return_rate KPI, выдано/возвращено/не-вернулось, график по неделям, разрез по точкам с именами + stale-флаг, фильтр периода.
- Локальный селектор проекта (cards-enabled из `config/cards.php`).
- Авторизация на точке: объектный гейт `isBoundToLocation` (cards.management ∨ смена ∨ Core-assignment).
- Аудит-трейл базовый: `performed_by/sent_by/accepted_by/returned_by/created_by`.
- Dev: `cards:seed-demo --fresh`, тест-аккаунты, вход по паролю.

- **A3. ✅ ЗАКРЫТ 2026-06-23. Контроль регулярности инвентаризации.** Команда `cards:inventory-watch` (суточный крон 05:00 МСК) шлёт `Log::warning cards.inventory.stale` по точкам без инвентаризации > `cards.inventory.stale_after_days` (10) и `cards.inventory.anomaly_open` по аномалиям; ручной прогон печатает таблицу, `--days=N` переопределяет порог. Тесты: +3.
